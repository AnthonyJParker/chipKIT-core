docs_template.html

This document is a template for creating new documentation pages for the chipKIT-core commit specific documentation. Copy this file, rename and populate with the content you want to document.

Document Naming and Location

GitHub pages does not support server side scripting. I would have liked to use PHP or TWIG to make a page template so that each specific document can be void of non-topic content as possible. The comprimise is to put all documents in the server root and prepend orginizing informaiton to the from of the file. I have used core_, api_ and docs_ so far.

menu.html

The menu.html file containg a bootstrap decorated list that is rendered as the responsive menu that appears on the top of each page. Edit this file to add a link to your newly created documentation.

bootstrap_head.html

The bootstrap.html is inclued in the header of each html file to try and keep documentation uncluttered.
